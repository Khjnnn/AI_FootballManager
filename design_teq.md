# 클로드 디자인 요청 가이드
### 배트맨 승무패 AI 분석 리포트 · 웹 뷰/대시보드 UI 제작용 (별첨 A)

**버전:** v1.0
**용도:** 리포트 HTML 웹 뷰, 회차 대시보드, 적중률 화면 등 본 프로젝트의 모든 화면을 클로드(Claude/Anthropic) 디자인 언어로 제작하도록 요청할 때 이 문서를 그대로 전달한다. 마지막 장의 프롬프트 템플릿은 Claude(또는 디자이너)에게 복사·붙여넣기로 사용한다.

---

## 1. 디자인 방향 요약

클로드 디자인 언어의 핵심은 "차분한 지성"이다. 어두운 배경에 네온 강조색을 쓰는 전형적인 스포츠 배팅 사이트의 문법(빨강/파랑 깜빡임, 과밀한 표, 긴박감 연출)을 의도적으로 피하고, 따뜻한 아이보리 바탕 위에 절제된 테라코타 포인트 하나로 신뢰감을 만든다. 이 프로젝트의 화면은 "배팅 사이트"가 아니라 "분석 리포트를 읽는 조용한 서재"처럼 보여야 한다. 수치는 밀도 있게, 장식은 최소로, 강조는 한 곳에만 쓴다.

## 2. 디자인 토큰

### 2.1 색상

| 토큰 | HEX | 용도 |
|---|---|---|
| `--bg` | `#FAF9F5` | 페이지 배경 (아이보리) |
| `--surface` | `#F0EEE6` | 카드·패널 배경 |
| `--surface-raised` | `#FFFFFF` | 강조 카드, 표 행 hover |
| `--border` | `#E3DFD3` | 헤어라인 구분선 (1px) |
| `--text` | `#1F1E1D` | 본문 텍스트 (웜 블랙) |
| `--text-muted` | `#6E6C64` | 보조 텍스트, 캡션, 수집 시각 |
| `--accent` | `#D97757` | 클로드 테라코타. 최종 픽, 핵심 버튼, 링크 등 **한 화면에 한 계열만** |
| `--accent-soft` | `#F5E5DE` | 강조 배경(픽 하이라이트 셀) |
| `--kraft` | `#D4A27F` | 보조 강조 (무승부 계열, 중립 지표) |
| `--olive` | `#6A6B5F` | 3차 계열 (패 계열, 비활성 지표) |
| `--positive` | `#5E7B5E` | 적중·상승 (채도 낮은 그린) |
| `--negative` | `#B0554B` | 미적중·하락 (채도 낮은 레드) |

승/무/패 데이터 시각화 고정 매핑: **승 = `--accent`(#D97757), 무 = `--kraft`(#D4A27F), 패 = `--olive`(#6A6B5F)**. 원색 빨강/파랑/초록 조합은 사용하지 않는다.

### 2.2 타이포그래피

클로드의 실제 서체(Styrene, Tiempos)는 라이선스 서체이므로 아래 대체 조합을 표준으로 한다. 한국어 화면 기준이다.

| 역할 | 서체 | 규격 |
|---|---|---|
| 디스플레이 (회차 제목, 팀명 대비) | **Noto Serif KR** (영문·숫자 Georgia 병용) | 28–40px, weight 600, letter-spacing -0.01em |
| 본문 (분석 서술) | **Pretendard** | 15–16px, line-height 1.7 |
| 데이터·숫자 (배당, 확률, 표) | **Pretendard** + `font-variant-numeric: tabular-nums` | 13–14px, 숫자 정렬 필수 |
| 캡션·메타 (수집 시각, 고지) | Pretendard | 12px, `--text-muted` |

세리프는 제목과 핵심 결론 문장에만 절제해서 쓰고, 표와 수치는 전부 산세리프 고정폭 숫자로 통일한다.

### 2.3 형태·간격·모션

모서리 라운딩은 카드 16px, 배지·태그 999px(필), 버튼 8px. 그림자는 `0 1px 3px rgba(31,30,29,0.06)` 수준의 아주 옅은 단계 하나만 사용하고, 구분은 그림자보다 1px 헤어라인 보더를 우선한다. 기본 간격 단위는 8px 배수(8/16/24/32/48). 모션은 hover 시 120ms ease 트랜지션과 리포트 로드 시 섹션 페이드인 한 번으로 제한하며, `prefers-reduced-motion`을 존중한다. 깜빡임·카운트다운 애니메이션·펄스 효과는 금지한다.

## 3. 컴포넌트 규칙

**회차 히어로.** 페이지 최상단. 세리프 대형 타이틀("승무패 38회차 · AI 최종 분석"), 그 아래 한 줄 메타(발매 마감 일시, 분석 기준 시각 "마감 12시간 전 · 07-12 08:30 수집"). 배경은 `--bg` 그대로, 장식 이미지 없이 타이포그래피가 히어로를 담당한다.

**경기 카드.** 14경기 각각 하나의 카드(`--surface`, 라운딩 16px). 상단에 리그·킥오프 시각(캡션), 중앙에 "홈팀 vs 원정팀"(세리프), 하단에 확률 바와 최종 픽 배지. 카드 전체가 경기 상세 리포트로 연결된다.

**승무패 확률 바.** 가로 스택 바 하나로 승/무/패 비율을 표현(승 `--accent`, 무 `--kraft`, 패 `--olive`). 각 구간 위에 % 라벨(tabular-nums). 이 확률 바가 본 서비스 화면의 **시그니처 요소**이며, 화면 전체에서 유일하게 색이 풍부한 부분이 되도록 주변을 조용하게 유지한다.

**최종 픽 배지.** 필 형태. 픽이 "승"이면 `--accent-soft` 배경 + `--accent` 텍스트. 보조 픽은 보더만 있는 아웃라인 배지로 위계를 낮춘다.

**신뢰도 배지.** 상/중/하 3단계 텍스트 배지. 색이 아닌 채움 정도로 구분(상: 채움, 중: 아웃라인, 하: 점선 아웃라인)하여 색맹 접근성을 확보한다.

**배당 비교 표.** 열: 배트맨 배당 | 해외 배당 | 내재확률 | 괴리. 괴리 열만 `--positive`/`--negative`로 부호 표시하고 나머지는 무채색. 행 구분은 헤어라인, 셀 패딩 12px, 숫자 우측 정렬.

**결장자 태그.** 회색 아웃라인 필 태그에 "선수명(사유)" 표기. 빨간 경고색을 쓰지 않는다 — 정보이지 경보가 아니다.

**고지 푸터.** 모든 화면 하단 고정 문구: "본 분석은 통계적 참고 자료이며 구매 결과를 보장하지 않습니다. 데이터 기준: {수집시각}." 12px, `--text-muted`.

## 4. 카피(문구) 톤

라벨은 시스템 용어가 아니라 사용자가 이해하는 말로 쓴다("데이터 페치 완료"가 아니라 "07-12 08:30 기준 최신 정보"). 문장형 대소문자·평서형을 쓰고 느낌표와 과장 표현("적중 보장", "필승 픽")은 금지한다. 버튼은 동작을 그대로 말한다("경기 분석 보기", "회차 리포트 내려받기"). 빈 화면과 오류는 방향을 제시한다("아직 이번 회차 분석이 생성되지 않았습니다. 마감 12시간 전에 자동 생성됩니다.").

## 5. Do / Don't

| Do | Don't |
|---|---|
| 아이보리 바탕 + 테라코타 포인트 한 계열 | 다크 배경 + 네온/원색 강조 |
| 헤어라인 보더로 구획, 여백으로 위계 | 짙은 그림자, 그라디언트 배경 |
| tabular-nums로 수치 정렬 | 비율 다른 숫자 폰트 혼용 |
| 확률 바 하나에 색을 집중 | 카드마다 다른 강조색 |
| 신뢰도를 형태(채움/아웃라인)로 구분 | 빨강=위험 식 색 의존 표기 |
| 절제된 세리프 제목 | 본문 전체 세리프, 장식 서체 |

## 6. 디자인 요청 프롬프트 템플릿 (복사해서 사용)

아래 템플릿을 Claude에게 그대로 붙여 넣고, `{ }` 부분만 채워 요청한다. 필요 시 이 가이드 문서 전체를 함께 첨부하면 정확도가 올라간다.

```
배트맨 승무패 AI 분석 리포트의 {회차 종합 리포트 / 경기 상세 / 적중률 대시보드} 화면을
클로드(Anthropic) 디자인 언어로 제작해줘. 단일 HTML 파일(내장 CSS)로 만들어줘.

[디자인 토큰 — 반드시 이 값 사용]
- 배경 #FAF9F5, 카드 #F0EEE6, 보더 #E3DFD3(1px 헤어라인)
- 텍스트 #1F1E1D, 보조 #6E6C64
- 강조(픽·버튼) #D97757, 강조 배경 #F5E5DE
- 승/무/패 시각화: 승 #D97757 · 무 #D4A27F · 패 #6A6B5F
- 적중 #5E7B5E, 미적중 #B0554B
- 제목: Noto Serif KR 600 / 본문·숫자: Pretendard, 숫자는 tabular-nums
- 카드 라운딩 16px, 배지 pill, 그림자 최소, 간격 8px 배수

[필수 컴포넌트]
1. 회차 히어로: 세리프 타이틀 + "마감 12시간 전 수집" 메타 한 줄
2. 경기 카드 14개: 리그/킥오프, 팀명(세리프), 승무패 스택 확률 바(% 라벨),
   최종 픽 배지(#F5E5DE 배경), 신뢰도 배지(상=채움/중=아웃라인/하=점선)
3. 배당 비교 표: 배트맨/해외/내재확률/괴리 4열, 숫자 우측 정렬, 괴리만 색 부호
4. 결장자 회색 아웃라인 태그
5. 하단 고지: "본 분석은 통계적 참고 자료이며 구매 결과를 보장하지 않습니다."

[금지]
다크 배경, 네온·원색 강조, 그라디언트, 깜빡임/펄스 애니메이션, 느낌표 카피,
"필승/보장" 류 문구, 카드마다 다른 강조색.

[분위기]
배팅 사이트가 아니라 차분한 분석 리포트. 색은 확률 바에만 모으고
나머지는 아이보리·웜그레이로 조용하게. 반응형(모바일 1열) 지원.

샘플 데이터: {회차 번호, 경기 2~3개 예시 데이터 붙여넣기}
```

## 7. 적용 체크리스트

납품물 검수 시 다음을 확인한다: 페이지 어디에도 #D97757 계열 외의 강조색이 없는가, 모든 수치가 tabular-nums로 정렬되는가, 승/무/패 색 매핑이 화면 전체에서 일관되는가, 신뢰도가 색 없이도(형태로) 구분되는가, 고지 문구와 데이터 수집 시각이 하단에 있는가, 모바일(390px)에서 카드가 1열로 정상 배치되는가, `prefers-reduced-motion` 설정 시 애니메이션이 꺼지는가.
